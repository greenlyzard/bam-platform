## CONFLICTS BETWEEN DOCS

- **Module count: 9 vs 12 vs 13.** `docs/CLAUDE.md` lists 9; root `CLAUDE.md` Section 7 lists 13; `docs/MODULES.md` lists 12; `docs/strategy/platform-product-requirements.md` lists 9. No single source of truth.
- **RBAC source-of-truth: `profiles.role` vs `profile_roles`.** `docs/ROLE_BASED_NAV_SPEC.md` says "Roles are stored in `profiles.role`" — root `CLAUDE.md` Section 4 says "Query `profile_roles` table, NOT `profiles.role`" and labels this the most common error.
- **Inbound email sender matching.** `docs/COMMUNICATIONS_INBOX.md` matches sender via `user_profiles.email`; newer `docs/CONTACT_ALIASES.md` overrides with `contact_channels` first → `families.billing_email` → `profiles.email`. INBOX uses a table name (`user_profiles`) that doesn't exist.
- **Substitute table names.** `docs/TEACHER_SUBSTITUTE_COVERAGE.md` defines singular `absence_record` and `substitute_assignment`; DB has plural `absence_records` and `substitute_requests`. Same conflict in `docs/TEACHER_ABSENCE_SUBSTITUTE_SUMMARY.md`.
- **Rate tables.** `docs/TEACHER_RATE_MANAGEMENT.md` defines `rate_definition`/`global_rate`/`teacher_rate`; live DB has only `teacher_rate_cards`.
- **Onboarding checklist tables.** `docs/SUBSTITUTE_TEACHER.md` defines `onboarding_templates`/`onboarding_checklists`; DB has neither — closest match is `teacher_compliance`.
- **Class group auto-create trigger.** `docs/COMMUNICATIONS.md` Section 1.2 ties auto-create to `classes.status='active'`; `docs/COMMUNICATIONS_HUB.md` says "per enrolled class" without specifying status field.
- **CLAUDE.md migration rules differ.** Root says `CREATE TABLE IF NOT EXISTS` AND `DROP POLICY IF EXISTS before CREATE POLICY`; docs/CLAUDE.md adds stricter rule "No forward FK references." Doesn't contradict but the docs/ rule is stricter.
- **Schedule UI canonical page.** `docs/UNIFIED_SCHEDULE.md` says merge `/admin/classes` into `/admin/schedule` (301 redirect); `docs/CLASSES.md` v2.0 still treats `/admin/classes` as the master surface. Either UNIFIED was implemented and CLASSES.md is stale, or it's aspirational.
- **Adult Pilates/Gyrotonic.** `docs/ENROLLMENT_QUIZ_SPEC.md` says these are "not in the standard class catalog" — but `docs/CLASSES.md` discipline list does not include Pilates/Gyrotonic at all, so quiz logic referencing them as a class option may misfire.
