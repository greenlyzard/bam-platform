## 1. Overview

Centralized document management for families and admin. Every document in the platform — signed contracts, waivers, policies, invoices, health records, admin uploads — lives here with consistent access control, expiry tracking, and a parent-facing to-do list for outstanding items.

Photos are NOT in this module — photos use Google Photos album links (per STUDENT_PROFILE.md Section 8).

## 3. Database Schema

```sql
CREATE TABLE IF NOT EXISTS family_documents (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id           uuid NOT NULL REFERENCES tenants(id),
  family_id           uuid REFERENCES families(id),
  student_id          uuid REFERENCES students(id),   -- null = family-level doc
  document_type       text NOT NULL
    CHECK (document_type IN (
      'contract','waiver','policy_acknowledgment','invoice',
      'health_record','registration','custom_upload','google_doc'
    )),
  title               text NOT NULL,
  description         text,

  -- Storage
  file_url            text,             -- Supabase Storage path (PDFs, uploads)
  external_url        text,             -- Google Doc link or external URL
  file_size_bytes     integer,

  -- Signing / acknowledgment
  requires_signature  boolean DEFAULT false,
  signed_at           timestamptz,
  signed_by           uuid REFERENCES profiles(id),
  signature_data      text,             -- base64 or typed name

  -- Expiry
  expires_at          date,             -- null = never expires
  expiry_reminder_sent_at timestamptz,

  -- Status
  status              text NOT NULL DEFAULT 'pending'
    CHECK (status IN (
      'pending',        -- requires action from parent
      'completed',      -- signed/acknowledged/paid
      'expired',        -- past expiry date
      'voided',         -- admin voided
      'uploaded'        -- admin-uploaded, no action required
    )),

  -- Visibility
  visible_to_parent   boolean NOT NULL DEFAULT true,
  visible_to_student  boolean NOT NULL DEFAULT false,  -- true for 18+

  -- Admin tracking
  uploaded_by         uuid REFERENCES profiles(id),    -- admin who uploaded
  uploaded_on_behalf  boolean DEFAULT false,            -- true = admin uploaded for family
  admin_notes         text,                            -- internal only

  -- Linking
  enrollment_id       uuid REFERENCES enrollments(id),
  season_id           uuid REFERENCES seasons(id),
  contract_template_id uuid,                           -- links to contract_templates

  created_at          timestamptz DEFAULT now(),
  updated_at          timestamptz DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_family_docs_tenant   ON family_documents(tenant_id);
CREATE INDEX IF NOT EXISTS idx_family_docs_family   ON family_documents(family_id);
CREATE INDEX IF NOT EXISTS idx_family_docs_student  ON family_documents(student_id);
CREATE INDEX IF NOT EXISTS idx_family_docs_status   ON family_documents(status);
CREATE INDEX IF NOT EXISTS idx_family_docs_type     ON family_documents(document_type);
CREATE INDEX IF NOT EXISTS idx_family_docs_expiry   ON family_documents(expires_at)
  WHERE expires_at IS NOT NULL;
```

## 5. Admin Document Library

### Route: `/admin/documents`

**Filters:**
- Family (search by name)
- Student
- Document type
- Status (pending / completed / expired / all)
- Season
- Expiry (expiring soon / expired / none)
- Uploaded by (system / admin)

**Table columns:**
- Family name
- Student name
- Document type badge
- Title
- Status badge
- Created date
- Expiry date (amber if within 30 days, red if expired)
- Actions: View / Download / Void / Send Reminder

**Bulk actions:**
- Send reminder to all families with pending items
- Export to CSV

### Admin Upload Flow

Admin can upload a document on behalf of a family:
1. Admin → Documents → Upload
2. Select family + student (optional)
3. Select document type
4. Upload file (PDF, DOCX, image) OR paste Google Doc URL
5. Set: title, description, visible to parent (toggle), requires signature (toggle), expiry date (optional)
6. Save → appears in family's document list
7. If `visible_to_parent = true` AND `requires_signature = true` → parent gets push notification

### Expiry Management

Nightly cron at 3am:
- Finds all documents where `expires_at <= now() + 30 days` AND `expiry_reminder_sent_at IS NULL`
- Sends reminder to family: "[Document] expires on [date] — please renew"
- Sets `expiry_reminder_sent_at = now()`
- Finds all documents where `expires_at < now()` AND `status != 'expired'`
- Updates status to `expired`
- Creates new `pending` document record for the renewal (same type, blank signature)

## 6. Parent Document Library

### Route: `/portal/documents`

**Two tabs:**

**"To Do" tab** — outstanding items only
- Sorted by urgency (overdue first, then by due date)
- Each item has a clear CTA: Sign / Pay / Complete / Acknowledge
- Completed items disappear from this tab

**"All Documents" tab** — complete history
- Filter by student, document type
- Signed/completed docs show: date completed, signed by
- Download PDF button on all completed docs
- Google Doc links open in new tab

### Student-level visibility

- Documents with `student_id` set are grouped under that student's name
- Family-level documents (no student_id) shown at the top
- Students 18+ can see their own documents if `visible_to_student = true`

## 7. Google Doc Integration

When document_type = `google_doc`:
- Admin pastes a Google Docs/Sheets/Drive share URL
- Platform stores the URL in `external_url`
- Parent sees a "View Document" button that opens the link in a new tab
- No signature collection for Google Docs — use for informational docs only
- For docs requiring signature: export to PDF first, then upload as `custom_upload`
